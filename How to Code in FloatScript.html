FloatScript Coding Guide

FloatScript is a versatile scripting language designed for various computational tasks, offering a rich set of commands and libraries for efficient programming.

Design Philosophy

  • Hybrid Execution Model (Interpretation, JIT, AOT)
  • Developer Empowerment and Control
  • Modularity and Extensibility
  • Readability and Maintainability
  • Performance and Efficiency
  • Error Handling and Debugging
  • Community and Ecosystem

Core Features

  • Dynamic and static typing with lex, let, float
  • Comprehensive output commands (std::out, println_, std::log)
  • Robust control flow structures
  • Multiple function definition styles (def, fn, init)
  • Advanced compilation (JIT and AOT)
  • Specialized libraries for various domains

General Commands

exit Command

Terminates the FloatScript interpreter gracefully, releasing allocated resources.

exit

The exit command orchestrates a graceful shutdown of the interpreter environment, including:

  • Releasing dynamically allocated memory
  • Closing open file handles
  • Terminating background processes/threads

./help Command

Displays a comprehensive list of available commands and their basic usage.

./help

The help output is categorized into sections:

  • General Commands
  • Variables & Assignment
  • Output
  • Control Flow & Blocks
  • Functions
  • JIT Compilation
  • AOT Compilation
  • Libraries

Comments

Single-line comments are denoted by //.

// This is a single-line comment
lex x = 10 // Assign 10 to x

Comments are vital for code documentation, allowing programmers to add explanatory notes, clarify complex logic, or temporarily disable lines of code without deleting them.

Variables & Assignment

lex Assignment

The most versatile assignment operator, allowing assignment of any data type.

lex my_str = 'hello' // String
lex count = 10 // Integer
lex price = 99.99 // Float
lex is_active = true // Boolean
lex my_array = [1, 2, 3] // Array
lex result = (a + b) * c // Expression

lex provides dynamic typing where the interpreter automatically infers the type based on the assigned value.

let Assignment

Explicitly assigns integer values to variables.

let count = 10
let index = 0
let max_users = 100

let ensures the assigned value conforms to the integer data type, preventing accidental assignment of non-integer values.

float Assignment

Explicitly assigns floating-point values to variables.

float pi = 3.14159
float gravity = 9.81
float temperature = 36.6

float helps maintain precision and ensures mathematical operations are performed with floating-point arithmetic.

Ternary Operator

Concise conditional expression for assignments.

lex max_val = x > y ? x : y
lex status = is_logged_in ? 'Online' : 'Offline'
lex config_value = user_input ? user_input : 'default'

The ternary operator (condition ? value_if_true : value_if_false) reduces the need for verbose if-else statements for simple conditional assignments.

Output Commands

std::out

Primary method for printing variables or literal values to standard output.

std::out my_variable
std::out "Hello, FloatScript!"
std::out "The value is: ", std::out my_var

std::out can accept any data type, automatically converting it to a string representation. It does not automatically append a newline.

std::out (i:v)

Prints all elements of array v except the element at index i.

lex my_array = [10, 20, 30, 40]
std::out (1:my_array) // Output: [10, 30, 40]

This selective printing capability offers fine-grained control over array output, useful for debugging array manipulations.

println_

Prints the value of a variable followed by a newline character.

println_result // Prints value of 'result' with newline
println_'Hello, world!' // Literal string with newline

println_ provides a concise way to quickly display variable contents in a readable format, ensuring each output appears on a new line.

std::log

Structured logging command that prefixes messages with [STD LOG].

std::log "Processing complete"
// Output: [STD LOG] Processing complete

std::log "[ERROR] File not found: config.txt"
// Output: [STD LOG] [ERROR] File not found: config.txt

std::log is invaluable for tracing program execution, recording events, and identifying potential issues during development and testing.

Control Flow & Blocks

if Statement

Conditional execution of a block of code.

if _i > 5_ {
  std::out "i is greater than 5"
}

The condition can be any expression that resolves to a boolean value, including comparisons, logical operations, or variable evaluations.

if _ ; _

Executes a block if a value exists in an array.

if _'apple'; my_list_ {
  std::out "Apple found!"
}

This syntax offers a clean and efficient way to check for the presence of an element within a given array.

while Loop

Repeatedly executes a block of code as long as a condition is true.

let i = 0
while _i < 10_ {
  std::out i
  i = i + 1
}

while loops are ideal for situations where the number of iterations is unknown and depends on a dynamic condition.

for Loop

Iterates through a sequence of values.

for (let i = 0; i < 10; i++) {
  std::out i
}
for (lex val in my_array) {
  std::out val
}

FloatScript supports both C-style for loops and for-each style loops for iterating over arrays and other iterable objects.

Functions

def Function

Defines a traditional, globally scoped function.

def my_function(param1, param2) {
  // Function body
  return param1 + param2
}

lex result = my_function(10, 20)

def functions are accessible from anywhere in the script, making them suitable for utility functions and core logic.

fn Function

Defines a function that is locally scoped to its block.

if _condition_ {
  fn local_func() {
    std::out "This is a local function"
  }
  local_func()
}

fn functions are useful for creating helper functions that are only needed within a specific control flow block, preventing pollution of the global namespace.

init Function

Defines a constructor function for a class or object.

class MyClass {
  init(name) {
    this.name = name
  }
}

lex my_obj = MyClass('example')

init is a special function that is automatically called when a new instance of an object is created, handling the initialization of its properties.

Compilation Commands

auto

Enables automatic Just-In-Time compilation for eligible functions.

auto
// Functions will be JIT-compiled on first call

The auto command optimizes the performance of frequently executed code paths by compiling them to native machine code at runtime.

jit::compile

Explicitly requests JIT compilation for a specific function.

def my_hot_func() { ... }
jit::compile my_hot_func

This command gives developers precise control over which functions should be JIT-compiled, bypassing the automatic detection process.

jit::run

Executes a JIT-compiled function.

jit::run my_compiled_func
// Executes the compiled version

jit::run is used to call a function that has already been JIT-compiled, ensuring the performance benefits are realized without re-interpretation.

aot::compile

Compiles the entire script to native code before execution.

aot::compile my_script.fs // Generates a binary file

AOT compilation is suitable for production environments where startup time and peak performance are critical, as the compilation step is performed offline.

aot::run

Executes a previously AOT-compiled binary.

aot::run my_binary
// Directly executes the native code

This command provides the highest level of performance, as the code is already in a machine-executable format, bypassing the need for interpretation or JIT compilation at runtime.

aot::dis

Disassembles a compiled binary file to view the underlying native code.

aot::dis my_binary
// Shows assembly code

aot::dis is an advanced debugging tool for developers who need to inspect the low-level output of the AOT compiler to optimize their code.

Libraries

Library:rpn

Reverse Polish Notation (RPN) library for stack-based calculations.

// Example: (5 + 3) * 2
lex result = .eval("5 3 + 2 *")

The library is a specialized tool for efficiently performing complex mathematical calculations using the RPN syntax, ideal for scientific and engineering applications.

Library:mem

Provides functions for low-level memory management and manipulation.

lex buffer = .alloc(1024)
.set(buffer, 0, 255)
.free(buffer)

The library gives advanced developers direct control over memory, enabling them to implement high-performance data structures and algorithms.

Library:fs

File system library for reading, writing, and managing files and directories.

lex content = .read("config.txt")
.write("output.txt", "Hello!")
lex exists = .exists("data/")

The library is a robust interface for interacting with the local file system, essential for scripts that need to persist data or process files.

Library:vect

Provides advanced vector and matrix operations for linear algebra and graphics.

lex vec_a = .new(1, 2, 3)
lex dot_prod = .dot(vec_a, vec_b)
lex cross_prod = .cross(vec_a, vec_b)

The library simplifies complex mathematical operations, making it an excellent choice for scientific computing, simulations, and game development.

Library:jit

Exposes JIT compilation APIs for programmatic control.

def fast_func() { ... }
.compile(fast_func)
.run(fast_func)

The library provides granular control over JIT compilation, allowing developers to dynamically compile and execute functions at runtime for optimal performance.

Library:io

Input/Output library for handling user input and external data streams.

lex name = .readln("Enter your name: ")
.write("Hello, " + name)

The library is the standard way to interact with users and other data sources, providing simple and powerful functions for both reading and writing.

Library:bit

Provides bitwise operations for low-level data manipulation.

lex a = 5 // 0101
lex b = 3 // 0011
lex c = .and(a, b) // 0001 (1)

The library is essential for tasks requiring fine-grained control over individual bits, such as embedded programming, cryptography, and network protocols.
