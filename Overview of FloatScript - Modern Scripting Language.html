Architecture Overview

Understanding the FloatScript execution model from source to execution.

FloatScript is a dynamically typed, interpreted scripting language designed for simplicity and ease of use. Its execution model relies on a virtual machine (VM) that processes a compact, efficient bytecode representation of the source code.

This architecture, encompassing a lexer, parser, compiler, and virtual machine, provides profound insights into compiler design. The addition of Ahead-of-Time (AOT) and Just-in-Time (JIT) compilation offers flexible performance tuning for any task.

The Execution Pipeline

  • 1. Lexical Analysis

    Source code is broken down into a stream of tokens (e.g., keywords, identifiers, operators).

  • 2. Parsing

    Tokens are transformed into an Abstract Syntax Tree (AST), representing the code's structure.

  • 3. Compilation

    The AST is traversed and converted into a series of compact bytecode instructions.

  • 4. Execution

    The bytecode is executed by the stack-based FloatScript Virtual Machine (VM).

factorial.flt
fn factorial {
    if _n == 0_ {
        .rt { 1 }
    }
    lex next_n = n - 1
    // Recursive call is complex; this is a conceptual example
    lex recursive_result = next_n // Pretend factorial(next_n) returns here
    .rt { n * recursive_result }
}

Bytecode Architecture

The efficient intermediate representation that powers the VM.

FloatScript programs are not executed directly; they are first compiled into an intermediate representation known as bytecode. This bytecode is a set of simple, platform-independent instructions that are executed by the FloatScript Virtual Machine (VM).

The instruction set is designed to be lean yet powerful, enabling efficient execution. The VM is a stack-based machine, meaning it performs operations by pushing values onto and popping them off a central data structure: the operand stack.

Opcode Name Description Operands
OP_CONSTANT Pushes a constant value from the constant pool onto the stack. constant_index (1 byte)
OP_ADD Pops two numbers from the stack, adds them, and pushes the result back. None
OP_SUBTRACT Pops two numbers, subtracts the top from the second, pushes the result. None
OP_MULTIPLY Pops two numbers, multiplies them, and pushes the result. None
OP_GET_GLOBAL Gets a global variable's value and pushes it onto the stack. constant_index (for var name)
OP_SET_GLOBAL Sets a global variable to the value at the top of the stack (does not pop). constant_index (for var name)
OP_PRINT Pops the top value from the stack and prints it to the console. None
OP_POP Pops and discards the top value from the stack. None
OP_RETURN Ends execution of the current function or program. None

AOT and JIT Compilation

Optimize performance with Ahead-of-Time and Just-in-Time compilation.

Ahead-of-Time (AOT)

The AOT compiler translates FloatScript source code into bytecode, which is then run by the VM. This is ideal for production environments where startup performance and predictability are key.

aot_example.flt
// Compile a program into bytecode
aot::compile my_program {
    lex a = 10
    lex b = 20
    lex c = a + b
    std::out c
}

// Run the compiled program
aot::run my_program
> 30

// Disassemble to view bytecode
aot::dis my_program
> --- Disassembly: my_program ---
> 0000 OP_CONSTANT 0 (10)
> 0002 OP_SET_GLOBAL 1 ('a')
> ... (and so on)

Just-in-Time (JIT)

JIT mode (simulated) allows for dynamic compilation of code blocks at runtime. This can provide performance boosts for frequently executed loops or functions within an interpreted session.

jit_example.flt
// Manually compile a routine
jit::compile heavy_math {
    let total = 0
    for i from 1 to 1000 {
        lex total = total + i
    }
    std::out total
}

// Run the compiled JIT routine
jit::run heavy_math
> 500500

// Enable auto-JIT for a block
auto 
if _true_ {
    // This block will be JIT-compiled
    std::out 'This is fast!'
}

Terminal Usage

Getting started with FloatScript in your terminal is simple and intuitive.

Running the Interpreter (REPL)

To run the interactive FloatScript shell (Read-Eval-Print Loop), simply execute the interpreter binary.

$ ./floatscript_interpreter
This is |floatscript.flt|...
> lex my_message = 'Hello, FloatScript!'
> std::out my_message
Hello, FloatScript!
> lex x = 10 + 5 * 2
> std::out x
20
> exit

This will show a prompt. You can also include libraries by typing #include <fs, mem>.

Running Script Files

To execute a FloatScript file (e.g., `myscript.flt`), pass it as an argument to the interpreter.

$ ./floatscript_interpreter myscript.flt
Executing myscript.flt...
Hello from a FloatScript file!
Calculation result: 42
Script executed successfully.

You may need to provide the full path to the interpreter, for example: /home/ubuntu/floatscript_interpreter myscript.flt

Performance Advantages

Why FloatScript stands out from other scripting languages.

Architectural Strengths

Stack-Based VM

Simplifies instruction logic, reduces register pressure, and leads to a smaller, more maintainable VM implementation.

Bytecode Compactness

A small, focused set of opcodes reduces memory footprint and improves cache performance during execution.

Direct Compilation

Moves directly from AST to bytecode without complex intermediate representations, speeding up the compilation phase.
